> Documentação: return: you only use the return statement in script functions and methods, and it is used
to return a value from the function to be used in further code or function calls. It should be noted that
the execution of the function ends at the return statement, meaning that any code which comes after 
return has been called will not be run.

Sistema de parentesco: um objeto filho herda todo o código do objeto pai. Isso é muito interessante porque
me poupa o trabalho de escrever todo o código de um inimigo novo do zero, quando o comportamento dele é
extremamente semelhante ao de outro inimigo que eu já codei.

Técnica de programação: 
Máquina de estado ou state machine.
A grosso modo, na programação orientada a objetos, é possível dividir um código entre vários estados. Aplicando
à programação de jogos, um mob ou npc pode tomar várias ações distintas, ou seja, ter vários estados, e chamamos
o tipo de código criado para programar isso de máquina de estado.
No GameMaker, é possível criar um máquina de estado usando switch.
